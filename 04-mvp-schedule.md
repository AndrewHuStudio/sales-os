## 第 1 周：数据底座 + 评分模型

### 1.1 开发目标
建立可用的数据底座 + 跑通 WorkBuddy 评分模型，输出 8 个商区 300 家企业的初步清单。

### 1.2 主要任务

| 序号 | 任务 | 责任方 | 估时 |
| - | - | - | - |
| 1.1 | 飞书多维表格建表（6 张表） | 销售 + 助理 | 0.5 天 |
| 1.2 | 录入 8 个深圳重点商区（含边界 GeoJSON） | 销售 | 0.5 天 |
| 1.3 | 采集 300 家企业（首期 50 家种子） | 销售 | 2 天 |
| 1.4 | 为每家企业填入：行业/规模/商区/地址/官网 | 销售 | 1 天 |
| 1.5 | 部署评分 Agent 到飞书智能字段 | 开发 | 0.5 天 |
| 1.6 | 批量计算 300 家企业 WorkBuddy 评分 | 开发 | 0.5 天 |
| 1.7 | 校验评分合理性，调整评分 Agent 提示词 | 销售 + 开发 | 1 天 |

### 1.3 交付物

| 交付物 | 验收标准 |
| - | - |
| 6 张飞书多维表格 | 表结构与任务二完全一致 |
| 8 个商区 GeoJSON | 边界可正常显示在地图上 |
| 300 家企业清单 | 字段完整度 ≥ 95% |
| 300 家企业评分 | 80≤A / 60-79 B / <60 C 分布合理（参考：15% / 35% / 50%） |
| 评分明细 | 每家可查看 8 维度评分 + 理由 |

### 1.4 验收标准

1. ✅ 8 个商区全部录入，地图可正常显示。
2. ✅ 300 家企业全部录入，字段完整度 ≥ 95%。
3. ✅ 100% 企业已生成 WorkBuddy 评分。
4. ✅ 评分等级分布合理：A 占 10-20%，B 占 30-40%，C 占 40-60%。
5. ✅ 每家企业可查看评分理由（≥ 100 字）。

### 1.5 风险点与应对

| 风险 | 概率 | 影响 | 应对 |
| - | - | - | - |
| 300 家企业采集速度慢 | 高 | 高 | 优先完成 50 家种子 + 公开数据爬取 + AI 批量补全 |
| 评分分布不合理 | 中 | 高 | 抽样 20 家人工校验，反向调权重 |
| GeoJSON 边界不准 | 中 | 中 | 用腾讯地图 / 高德地图行政区边界代替，标注"近似边界" |
| 飞书多维表格性能瓶颈 | 中 | 中 | 300 家以内无压力；超过 1000 家考虑切换 PostgreSQL |

让销售每天打开系统，**3 分钟内**能定位到 10 家今天该打的客户，并看到每家的作战卡。

| 2.3 | 企业点位渲染（按评分着色） | 开发 | 0.5 天 |

| 2.5 | 企业列表页面（表格 + 6 筛选器 + 排序） | 开发 | 1 天 |

| 2.8 | 企业详情页 - KP 路线 Tab（调用 Agent） | 开发 | 0.5 天 |
| 2.9 | 企业详情页 - 触达话术 Tab（调用 Agent） | 开发 | 0.5 天 |

### 2.3 交付物

| 交付物 | 验收标准 |

5. ✅ 任意企业可一键复制触达话术。

### 2.5 风险点与应对

| 风险 | 概率 | 影响 | 应对 |
| - | - | - | - |
| 地图组件加载慢 | 中 | 中 | 懒加载 + 矢量切片；MVT 格式优先 |

| 多 Tab 切换性能 | 低 | 中 | 预加载 + 状态保留 |
| 数据权限 | 低 | 高 | MVP 阶段单机版，可后续加登录 |

| 风险 | 概率 | 影响 | 应对 |

| 飞书 API 限流 | 中 | 中 | 错峰 + 队列；必要时切换企业微信 |

## 上线标准（M4 必达）

### 必达指标
1. 销售连续 5 天主动打开系统。
2. 系统累计录入跟进记录 ≥ 50 条。
3. 至少 3 家企业进入"需求沟通"或更后阶段。
4. 至少 1 家企业进入 POC 阶段。
5. 销售对系统整体满意度 ≥ 7/10。

### 推迟上线条件
- W3 结束前未跑通跟进闭环 → 推迟到 W5。
- 评分模型认可度 < 60% → 推迟到 W5，并优先优化评分。
- AI 关键 Agent（跟进总结 / 话术）失败率 > 30% → 推迟到 W5。
