# 任务一 · WorkBuddy GeoSales OS 系统原型结构

> 版本：V1.0｜面向 MVP 落地｜先定原型，再动代码
>
> 7 个页面 × 5 个维度：页面目标 / 页面模块 / 字段信息 / 用户操作 / 跳转逻辑

---

## 全局导航与跳转关系图

```
                        ┌──────────────────┐
                        │  P1 首页工作台      │  ← 日常入口
                        └────────┬─────────┘
              ┌──────────┬──────┴──────┬──────────────┐
              ▼          ▼             ▼              ▼
       ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
       │ P2 深圳地图 │ │ P3 企业列表│ │ P6 销售漏斗│ │ P7 销售复盘│
       └────┬─────┘ └────┬─────┘ └────┬─────┘ └──────────┘
            │            │            │
            └──────┬─────┘            │
                   ▼                  │
            ┌──────────────┐          │
            │ P4 企业详情/   │◀─────────┘
            │   客户作战卡   │
            └──────┬───────┘
                   │
                   ▼
            ┌──────────────┐
            │ P5 跟进记录    │  ← 单条跟进详情（也可作 P4 内嵌 Tab）
            └──────────────┘
```

**跳转原则**：
- **首页工作台**是销售每日入口，跳转频次最高。
- **企业列表（P3）**是流量中转站（地图筛选 → 列表 → 详情）。
- **客户作战卡（P4）**是核心操作页，从地图、列表、漏斗都能进入。
- **跟进记录（P5）**可作为 P4 的内嵌 Tab，也可独立访问。

## 页面 1：首页工作台（Daily Cockpit）

### 1.1 页面目标
让销售每天打开系统 **30 秒内** 知道：今天该打谁、该见谁、该推谁。把"打开系统找行动项"的成本压到最低。

### 1.2 页面模块

| 优先级 | 模块 | 数据来源 | 展示形态 |
| - | - | - | - |
| ★★★ | 今日 Top 10 优先客户 | workbuddy_score≥80 & 今日无跟进 | 卡片列表，每张含：企业名、推荐 KP、一句话切入、行动按钮 |
| ★★★ | 今日待跟进 | next_followup_date = 今天 & 阶段 < 已成交 | 时间轴 + 状态色 |
| ★★☆ | 本周漏斗快照 | opportunities 表按 stage 聚合 | 横向漏斗条 + 数字 |
| ★★☆ | 高潜商区排名 | business_districts.priority = A | Top 5 商区卡片，含企业数 / A 类客户数 |
| ★☆☆ | 最近新增高潜 | companies 入库时间 < 7 天 & A 类 | 时间倒序列表 |
| ★☆☆ | 本周动作统计 | 触达 / 会议 / POC 三个计数器 | 仪表盘式数字 + 同比箭头 |
| ★★☆ | 系统推荐下一步 | 跟进总结 Agent 输出 | 单条强提示 + 跳转按钮 |

### 1.3 字段信息（核心卡片字段）

**今日 Top 10 优先客户卡片**

| 字段 | 字段来源 | 备注 |
| - | - | - |
| 企业名 | companies.name | |
| 所属商区 | business_districts.name | |
| 评分 | companies.workbuddy_score | 颜色编码：≥80 绿 / 60-79 黄 / <60 灰 |
| 推荐 KP | contacts[0].role | 第一优先 KP 角色 |
| 一句话切入 | AI 生成（基于企业画像 + 推荐场景） | 不超过 30 字 |
| 行动按钮 | 微信话术 / 拨打电话 / 标记已跟进 | |

### 1.4 用户操作

| 操作 | 行为 | 跳转 |
| - | - | - |
| 点击客户卡片 | 展开预览 | 内嵌抽屉 → 可选进入 P4 |
| 点击「生成话术」 | 触发话术 Agent | 弹出话术浮层，可一键复制 |
| 点击「标记已跟进」 | 跳转 P5 新增跟进 | |
| 点击「查看地图」 | 跳转 P2，地图自动定位到该商区 | 携带 business_district_id |
| 切换日期 | 切换到「明日 / 本周 / 逾期」视图 | 仅前端过滤 |
| 漏斗点击某阶段 | 跳转 P6，筛选该阶段客户 | 携带 stage |

### 1.5 跳转逻辑

- 接收外部参数：`?focus_company_id=xxx` 时高亮指定企业。
- 跳转出口：P2 / P3 / P4 / P5 / P6。
- 未登录态：跳转登录页（MVP 阶段可豁免，单机版无需登录）。

## 页面 2：深圳客户地图（Geo Map）

### 2.1 页面目标
让销售在地图上**一眼看到**深圳重点商区里 WorkBuddy 高潜客户的密度分布，迅速判断"哪一片值得今天重点扫一遍"。

### 2.2 页面模块

| 优先级 | 模块 | 说明 |
| - | - | - |
| ★★★ | 深圳行政区底图 | 浅色底图，南山/福田/宝安/龙华/龙岗五大区 |
| ★★★ | 8 个重点商区边界 | 多边形覆盖，配色按 priority（A 橙 / B 蓝 / C 灰） |
| ★★★ | 企业点位 | 按 workbuddy_score 着色 |
| ★★☆ | 商区热力图 | 切换图层，按 A 类客户数渲染 |
| ★★☆ | 筛选侧栏 | 行业 / 评分 / 状态 / 商区 |
| ★☆☆ | 商区详情弹窗 | 点击商区弹出，含统计指标 |
| ★☆☆ | 图例与图层切换 | 浮窗按钮 |

### 2.3 字段信息

**商区多边形属性**

| 字段 | 数据来源 | 用途 |
| - | - | - |
| name | business_districts.name | 标签显示 |
| company_count | business_districts.company_count | 商区卡片数字 |
| a_class_count | business_districts.high_score_count | 高潜数字 |
| priority | business_districts.priority | 多边形填充色 |

**企业点位属性**

| 字段 | 数据来源 | 用途 |
| - | - | - |
| 名称 | companies.name | hover tooltip |
| 评分 | companies.workbuddy_score | 点位颜色 |
| 阶段 | companies.current_stage | 点位边框样式 |
| 行业 | companies.industry | 行业筛选后保留 |

**筛选侧栏字段**

| 字段 | 类型 | 默认值 |

| 评分区间 | 区间滑块 | 0–100 |
| 客户等级 | 多选 A/B/C | A+B |
| 跟进状态 | 多选 | 全选 |
| 仅 A 类高潜 | 开关 | 关 |

### 2.4 用户操作

| 操作 | 行为 | 跳转 |
| - | - | - |
| 点击商区 | 弹出商区详情卡 | 弹层 + 「进入商区企业列表」按钮 |
| 点击企业点位 | 弹出企业 mini 卡片 | 弹层 + 「查看作战卡」按钮 |
| 调整筛选 | 实时刷新地图点位 | 无跳转 |
| 切换热力图层 | 替换点位显示 | 无跳转 |
| 放大/缩小/拖动 | 地图标准操作 | 无跳转 |

### 2.5 跳转逻辑

- 点击「进入商区企业列表」→ 跳转 P3，携带 `?district_id=xxx`。
- 点击「查看作战卡」→ 跳转 P4，携带 `?company_id=xxx`。
- 地图状态（缩放、中心）写入 URL query，刷新可还原。

## 页面 3：企业列表（Company Roster）

### 3.1 页面目标
作为地图、漏斗、复盘的"中转站"，让销售可以**集中筛选、批量处理**（标记跟进、导出 Excel、调整阶段）。

### 3.2 页面模块

| 优先级 | 模块 | 说明 |
| - | - | - |
| ★★★ | 筛选条件栏 | 商区/行业/等级/评分/状态/跟进日期 |
| ★★★ | 列表主表 | 字段见下表 |
| ★★☆ | 批量操作工具栏 | 标记跟进 / 导出 / 修改阶段 |
| ★★☆ | 列头排序 | 评分、规模、跟进时间 |
| ★☆☆ | 分页 / 每页条数 | 默认 50 |

### 3.3 字段信息

| 字段 | 字段名 | 字段来源 | 是否可排序 | 备注 |
| - | - | - | - | - |
| 公司名称 | name | companies.name | 否 | 链接到 P4 |
| 所属商区 | district | business_districts.name | 是 | |
| 行业 | industry | companies.industry | 是 | |
| 规模 | size | companies.company_size | 是 | 员工数映射到 S/M/L/XL |
| WorkBuddy 评分 | score | companies.workbuddy_score | 是 | 颜色编码 |
| 客户等级 | level | companies.customer_level | 是 | A/B/C 徽标 |
| 推荐场景 | scenarios | companies.recommended_scenarios | 否 | 多个用逗号分隔 |
| 当前状态 | stage | companies.current_stage | 是 | 阶段色 |
| 下一步动作 | next_action | companies.next_action | 否 | AI 建议 |
| 下次跟进时间 | next_followup_date | companies.next_followup_date | 是 | 逾期红色 |
| 操作 | - | - | - | 作战卡 / 跟进 / 话术 三个按钮 |

### 3.4 用户操作

| 操作 | 行为 | 跳转 |
| - | - | - |
| 点击公司名 | 跳转企业详情 | P4，携带 company_id |
| 点击「作战卡」 | 跳转作战卡 | P4 |
| 点击「跟进」 | 跳转新增跟进 | P5，携带 company_id |
| 点击「话术」 | 弹出话术浮层 | 内嵌对话框 |
| 勾选多行 | 激活批量工具栏 | 无跳转 |
| 点击「导出」 | 下载 Excel | 文件下载 |
| 修改阶段 | 下拉切换 | 落库后行内刷新 |

### 3.5 跳转逻辑

- 接收参数：`?district_id / ?industry / ?level / ?stage / ?min_score` 自动应用筛选。
- 列表状态可导出 URL，分享给同事或写日报。

| ★★★ | 概览 | 基础信息 + 评分 + 评分理由 | 评分 Agent |

| ★★★ | 跟进历史 | 该企业所有跟进记录倒序 | 跟进记录表 |
| ★★☆ | 机会 | 关联的机会列表 | opportunities 表 |
| ★★☆ | 备注 | 销售本人备注 | companies.notes |

| 操作 | 行为 |

- 「在地图查看」→ 跳转 P2，定位该企业点位。

## 页面 5：跟进记录（Followup Log）

### 5.1 页面目标
让销售在**1 分钟内**完成一次跟进记录（自然语言输入 + AI 自动结构化），并自动生成下一步动作。

### 5.2 页面模块

| 优先级 | 模块 | 说明 |
| - | - | - |
| ★★★ | 新增跟进表单 | 自然语言输入框 + 提交 |
| ★★★ | AI 结构化结果预览 | 提交后弹出，确认后落库 |
| ★★☆ | 该企业跟进历史 | 时间倒序 |
| ★★☆ | 销售漏斗 mini 视图 | 当前阶段 + 阶段历史 |
| ★★☆ | 快捷操作栏 | 约下次 / 调整阶段 / 标记输单 |

### 5.3 字段信息

**输入字段**

| 字段 | 类型 | 必填 | 备注 |
| - | - | - | - |
| 企业 | 公司选择器 | 是 | 默认从 P4 带入 |
| 沟通时间 | datetime | 是 | 默认当前 |
| 沟通对象角色 | 单选 + 自由输入 | 是 | CEO/COO/销售/市场/IT/HR/采购/其他 |
| 沟通对象姓名 | text | 否 | 可空（合规） |
| 沟通方式 | 单选 | 是 | 电话/微信/会议/邮件/线下 |
| 跟进摘要 | textarea | 是 | 自然语言，200–500 字 |

**AI 自动结构化输出**

| 字段 | 来源 Agent |
| - | - |
| 客户关注点 | 跟进总结 Agent |
| 客户异议 | 跟进总结 Agent |
| 客户当前意向 | 跟进总结 Agent |
| 销售阶段 | 跟进总结 Agent |
| 下一步动作 | 跟进总结 Agent |
| 下次跟进时间 | 跟进总结 Agent（建议 3/7/14 天） |
| 推荐材料 | 跟进总结 Agent |

### 5.4 用户操作

| 操作 | 行为 |
| - | - |
| 提交跟进 | 调用跟进总结 Agent，弹出结构化结果 |
| 编辑 AI 结果 | 任意字段可手动调整 |
| 「确认落库」 | 写入 followups 表 + 同步更新 companies.current_stage / next_action |
| 「约下次跟进」 | 触发企业微信 / 飞书日历提醒 |
| 「标记阶段为输单」 | 二次确认后写入 |
| 「返回企业详情」 | 跳转 P4，company_id 保留 |

### 5.5 跳转逻辑

- 接收参数：`?company_id=xxx` 可选。
- 无 company_id 时，企业字段改为必填公司选择器。
- 提交成功后默认跳回 P4，停留 2 秒后跳转 P3 下一条「待跟进」客户（可配置）。

## 页面 6：销售漏斗（Sales Funnel）

### 6.1 页面目标
让销售**一眼看清**自己手里客户分布在哪个阶段，识别卡点（如"已沟通 30 家但只进入演示 5 家"），决定本周冲刺哪个阶段。

### 6.2 页面模块

| 优先级 | 模块 | 说明 |
| - | - | - |
| ★★★ | 11 阶段漏斗图 | 横向漏斗 + 客户数 + 转化率 |
| ★★★ | 阶段客户列表 | 点击阶段展开 |
| ★★☆ | 阶段金额预估 | 按 opportunity 汇总 |
| ★★☆ | 阶段停留时长 | 平均停留天数 |
| ★★☆ | 风险标记 | 超过 N 天未跟进的客户标红 |

**漏斗阶段（11 阶段）**

| - | - | - | - | - | - |
| 1 | 未触达 | | - | - | 列表 |
| 2 | 已触达 | | | | 列表 |
| 3 | 有效回复 | | | | 列表 |
| 4 | 需求沟通 | | | | 列表 |
