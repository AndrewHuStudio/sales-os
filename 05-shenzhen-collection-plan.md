## 4. 采集流程

### 4.1 整体流程

```
渠道选择 → 批量初筛 → 人工补全 → 数据清洗 → 评分入库 → 抽样校验
   │          │           │           │           │           │
   ▼          ▼           ▼           ▼           ▼           ▼
  决定     1000 → 300   填必填项   查重/纠错   跑评分 Agent  抽 30 家
  从哪找   粗筛到精筛   +评分字段  +统一命名   自动出分    人工校验
```

### 4.2 7 个采集渠道

| 渠道 | 适用 | 优势 | 局限 |
| - | - | - | - |
| 企查查 | 全行业 | 数据齐全、已认证 | 部分企业参保人数为 0 |
| 高德/百度地图 | 园区企业 | 精准定位、楼栋号清晰 | 缺少规模/行业 |
| 官网 | 中大型企业 | 业务介绍、案例 | 需逐家访问 |
| 招聘网站 | 增长期企业 | 数字化/AI 信号 | 信号延迟 |
| 36 氪/虎嗅 | 科技/金融 | 融资/产品/团队信息 | 仅头部企业 |
| 微信公众号 | 跨境/品牌 | 业务动态、KP 发言 | 需关注公众号 |
| 公开演讲/活动 | 总部企业 | 高管身份、业务方向 | 信息碎片化 |

### 4.3 采集 SOP（单家企业）

| 步骤 | 动作 | 耗时 | 输出 |
| - | - | -: | - |
| 1 | 企查查搜索 → 获取工商信息 | 30s | 公司名/地址/成立时间/规模 |
| 2 | 高德地图确认办公地址 | 30s | 精确楼栋 + 商区归属 |
| 3 | 访问官网 → 抓取业务介绍 | 1min | 业务简介/细分行业 |
| 4 | BOSS 直聘 → 招聘 JD 关键词 | 1min | AI/数字化/腾讯生态信号 |
| 5 | 微信搜一搜 → 最近新闻 | 30s | 业务增长信号 |
| 6 | 填入采集表 → 触发 AI 评分 | 30s | 评分 + 评分理由 |
| 7 | 抽样校验（如有疑问） | 1min | 调整 / 标注"待核实" |

**单家耗时**：约 4 分钟  
**单人 8 小时可采集**：约 100 家  
**首期 300 家需投入**：3 人 × 1 天 或 1 人 × 3 天

## 6. 评分方式说明

### 6.1 评分触发

| 触发方式 | 时机 |
| - | - |
| 单条触发 | 录入一条企业后自动调用评分 Agent |
| 批量触发 | 每日 23:00 跑批，更新全部企业评分 |
| 手动触发 | 销售点击"重新评分"按钮 |

### 6.2 评分输出

| 输出 | 字段 |
| - | - |
| 总分 | companies.workbuddy_score |
| 8 维度明细 | companies.scoring_breakdown (JSON) |
| 评分理由 | companies.score_reasoning |
| 客户等级 | companies.customer_level（A/B/C 自动） |
| 推荐场景 | companies.recommended_scenarios |
| 可能痛点 | companies.pain_points |

### 6.3 评分校准

| 校准动作 | 时机 | 责任 |
| - | - | - |
| 抽 30 家人工评分对比 | W1 结束 | 销售 |
| 校准评分 Agent 提示词 | W2 周一 | 销售 + 开发 |
| 月度重新校准 | 每月最后一周 | 销售 |
| 基于转化数据反哺 | 每月复盘后 | 销售 + 数据 |
